Software

We prefer to keep our software stack as light as possible, so we will mainly use Java and JavaScript with some tools to support the languages. Use the OS you want.

  • Java

    Java is an object-oriented programming language which runs on the virtual JVM platform on any OS. A basic understanding of OO such as the 4 pillars of OO and using basic data structures such as Arrays and Maps is recommended. Look up some free Java learning resources between workshops if there are concepts you feel unsure of.

  • Maven

    Maven is a build and infrastructure tool. It solves tedious tasks such as manging your dependencies (it'll automatically download and resolve any thirdparty dependency you list in POM.xml which is Maven's manifest file), compiling your code, run your tests, and package your application for deployment.

    All Java projects we make for our workshops are made using Maven. This will give us a standardized way to run our code and tests, and it means attendees only need to learn this one build tool (others are Gradle or Ant, for example). In other words: learn Maven!

    Please read the getting started resources listed further down on this page and make a hello world Maven project in your editor, be it Eclipse, Vim, IntelliJ or something else (we recommend IntelliJ Community Edition).

    Make sure you have a working Maven installation and development environment (you can compile code and run tests) before the workshops!

  • Git

    Git is a version control system (VCS) which is extremely popular these days. We use Git for checking code into a repository when working on a project, especially when working in teams. There is a learning curve, but getting started with the basics should be easy enough. We also have a Nerdschool workshop on Git.

    The popular GitHub site (think Facebook for your git repositories) has a lot of excellent Git documentation and learning material. Take 15 minutes and go through their interactive git tutorial for understanding the basics and check out their other documentation resources. There is also some very nice cheat sheets for Git online, and while you should print one out and stick it near your monitor, this interactive cheat sheet is really good to have bookmarked.

    A visual Git client such as SourceTree can be nice when getting started.

  • IntelliJ Community Edition

    IntelliJ is an IDE which can handle many programming languages and tools, but really excels at Java. IntelliJ is a modern, elegant, and intuitive editor with clever refactorings and lots of helpful tools under the cover to help you with some tedious tasks.

    The Community Edition is free, and we recommend using this edition.

    If you prefer any other editor such as Eclipse, please feel free to use the editor of your choise, but make sure your environment and configuration with Java and Maven works before the workshops. Learn how to open a Maven project in your editor and how to compile and run tests.

  • An editor that understands JavaScript such as Atom, Visual Studio Code, or Sublime Text. IntelliJ as mentioned above also works well (and many others).

Make sure your development environment and tools works

Please make sure your Java installation works and that your Java class paths, SDK, and dependencies are set-up in the editor of your choise before the workshop! We only have 2.5 hours of actual work each session, so the less time spent fixing Java and missing/partial installs the better for everyone's sanity.

  • Using your editor, create a new hello world project and ensure you can build/compile it using Maven.
  • Also ensure that you can build and run a single unit test using jUnit. The test doesn't have to do anything, just build and run successfully.

How to ensure Java is correctly installed

  • Typing java -version or just java in a terminal should not produce an error
  • If you do get an error, ensure your JDK bin folder is on your machine's PATH environment variable.
    • Windows:
      • setting the PATH
      • Path to Java (or the JAVA_HOME env. variable) is typically C:\Program Files\Java\jdk1.8.0_60\bin or similar depending on your version.
